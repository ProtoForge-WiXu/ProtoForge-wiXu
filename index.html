Instrukcja obsługi aplikacji
Jak zainstalować, jak tworzyć protokoły, jak działa synchronizacja i wszystko inne
📱 Instalacja aplikacji (PWA)
Aplikacja działa w przeglądarce i instaluje się jak normalna apka — bez App Store.
🤖 Android (Chrome)
1. Otwórz Chrome → wejdź na adres apki
2. Menu ⋮ → "Dodaj do ekranu głównego"
3. Potwierdź → ikona pojawi się na pulpicie
4. Apka działa offline po instalacji
🍎 iPhone (Safari)
1. Otwórz Safari → wejdź na adres apki
2. Przycisk Udostępnij □↑ → "Dodaj do ekranu"
3. Kliknij Dodaj w prawym rogu
4. Apka gotowa na pulpicie

📋 Tworzenie protokołu — krok po kroku
1Kliknij 📋 NOWY na dolnym pasku → wybierz typ instalacji (SSP, DSO, DSG, Wentylacja ppoż, Drzwi ppoż…)
2Wpisz nazwę obiektu — po pierwszym razie apka zapamiętuje i podpowiada przy kolejnych protokołach.
3Wypełnij listę kontrolną — zaznacz wykonane czynności (✅ OK / ⚠️ Uwagi). Niezaznaczone nie pojawiają się w PDF.
4Wpisz usterki, zalecenia i uwagi → kliknij Zapisz (💾) lub PDF (generuje się lokalnie bez internetu).
5Po zapisaniu kliknij 🔄 Sync w górnym pasku żeby wysłać protokół do Google Sheets.
📅 Harmonogram serwisów
Kliknij 📅 HARMONOGRAM na dolnym pasku — widok miesięczny z siatką dni roboczych.

Dodaj serwis: kliknij komórkę dnia → dodaj obiekt → przypisz serwisanta i samochód.
Z protokołu: otwórz protokół → kliknij "Zaplanuj serwis" → data uzupełni się automatycznie z pola "Następny przegląd".
Kopiowanie: Ctrl+klik = zaznacz wiele → skopiuj/przenieś do innego dnia. Strzałki ◀▶ kopiują do sąsiedniego dnia roboczego.
🔄 Synchronizacja z Google Sheets
Apka działa offline — dane zapisywane są lokalnie na urządzeniu. Internet potrzebny jest tylko do synchronizacji.

Kiedy synchronizować: po każdym nowym protokole, po zmianach w harmonogramie, na początku dnia żeby pobrać zmiany innych urządzeń.

Kliknij 🔄 Sync w górnym pasku → poczekaj na komunikat "Synchronizacja zakończona".
Przycisk 📊 Excel otwiera arkusz Google Sheets bezpośrednio w przeglądarce.
🚀 Wdrożenie na GitHub Pages — szczegółowo
1Wejdź na github.com → zaloguj się lub zarejestruj (darmowe).
2Nowe repozytorium → nazwa (np. "strażak-protokoly") → Public → bez README → Create repository.
3Kliknij "uploading an existing file" → przeciągnij wszystkie 5 plików z ZIP-a → Commit changes.
4Settings → Pages → Source: Deploy from a branch → Branch: main → folder: / (root) → Save.
5Po chwili (1-2 min) apka będzie pod adresem: [login].github.io/[nazwa-repo] — link pod ktorym piszemy protokoły.
6Gdy wdrożysz GAS (URL z kroku 0) → otwórz apkę → ⚙️ Ustawienia → Adres serwera → wklej URL GAS → Zapisz.
❓ Najczęstsze pytania
Sync nie działa / "Błąd połączenia"
Sprawdź internet → upewnij się że URL GAS jest wklejony w Ustawienia → Adres serwera. GAS musi być wdrożony jako "Wszyscy".
Zapomniałem PIN-u
Wgraj ponownie index.html na github wtedy wróci fabryczny pin lub skontaktuj sie ze mna
Jak zaktualizować aplikację do nowej wersji?
⚙️ Ustawienia → "Sprawdź aktualizacje" → Zainstaluj. Lub wrzuć nowy index.html na GitHub — apka zaktualizuje się automatycznie przy następnym uruchomieniu online.
Biały ekran / apka nie ładuje się
Zamknij i otwórz ponownie. Jeśli nie pomaga: Chrome → Ustawienia → Aplikacje → [nazwa apki] → Wyczyść pamięć. Dane wróćą przy synchronizacji.
